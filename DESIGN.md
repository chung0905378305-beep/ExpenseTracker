# 记账本 iOS App — 前端设计系统（DESIGN v3 · 排版定稿）

> 配套需求 v3.4。本版锁定**界面排版与信息架构**。视觉：精致扁平 + 质感；导航：底 Tab(4) + 顶分段；全局无标题栏。

## 1. 全局排版规则（v3 定稿）
- **无标题栏**：内容顶到状态栏下方，不放大标题；页面身份靠顶部分段/首屏 Pad 自身表达。
- **内缩卡片留白**：内容左右留 16pt，卡片圆角 16、轻投影出层次。
- **列表行**：左 40pt 圆角图标块 → 分类(主,17) + 备注(副,13 次级) → 右 金额(tab 数字,红/绿)，日期/账户作次级小字附在备注下或金额下。
- **表单**：统一底部 Sheet（顶 20 圆角 + 拖拽把手 + 背景模糊），字段竖排带标签，保存置底。
- **搜索**：明细页顶部搜索条，输入即在当前列表**实时筛选**（不跳页）。
- **空状态**：居中大图标(56) + 标题 + 说明 + 主按钮。

## 5. 组件规范（v3 增补）
- **底 Tab Bar**：4 项（明细/统计/资产/我的），SF Symbol+文字，选中态 accent，角标显「待确认+续费」数。
- **记账入口**：底 Tab 无「记账」页；记一笔改为点明细页右下「+」浮钮打开**底部 Sheet**（顶 20 圆角 + 拖拽把手 + 模糊背景），取消按钮在 Sheet 左上。
- **记一笔 Pad（Sheet）**：竖向堆叠——类型分段 → 金额(居中,语义色) → 分类 5 列图标网格 → 账户/日期/备注字段 → 底部自定义数字键盘(含绿色保存)。无标题栏。
- **可拖拽卡片（统计页）**：每张卡右上角拖拽手柄(⠿)，长按进入编辑态可上下重排；卡片含：订阅卡、月度汇总卡、分类占比卡、趋势卡、年度卡。顺序用户自定义，设置可隐藏某卡。
- **详情页**：点账单行 Push 进交易详情（展示全字段 + 关联退款 + 编辑/删除）。
- **底部 Sheet**：转账/加订阅/加持仓/加账户/编辑交易统一用。
- **Chart**：占比=横向条形；趋势=折线(近6月)；年度=柱状。Swift Charts，取主题色。
- **卡片 / 按钮 / Toggle / Search / Badge / Empty**：见 v2。

## 6. 导航与信息架构（v5 终）
- **底 Tab（4）**：① 明细 ② 统计 ③ 资产 ④ 我的
- **明细 Tab（首屏）**：搜索条(置顶) → 月度汇总卡(收入绿/支出红/结余) → 按月分组紧凑列表；点行→详情页；右下悬浮「+」→ 打开记一笔底部 Sheet。
- **记账（记一笔 Sheet）**：非 Tab，由明细页「+」浮钮唤起。竖向堆叠 Pad（类型分段 → 金额居中 → 分类 5 列网格 → 账户/日期/备注 → 底部数字键盘）；Sheet 左上「取消」。
- **统计 Tab（可定制仪表盘）**：顶部时间范围分段**新增「周」→ [周|本月|本年|自定义]**，默认「本周」；其余不变：可拖拽重排的卡片流，首卡为「我的订阅」卡(显示最近续费日+笔数，点进订阅页)；设置可隐藏订阅卡。卡片：月度汇总/分类占比/趋势/年度/订阅。点分类卡可下钻明细。
- **订阅页（从统计订阅卡进入）**：顶分段[即将续费|全部] + 即将续费卡(按天数) + 列表(三模式标签) + 右下+。
- **资产 Tab（单页滚动，按账户类别分区）**：顶部 净资产 hero 卡(总资产/总负债/净资产 + 迷你走势) → 分区：① 投资账户（每卡显「总计」市值/成本/盈亏% + 嵌套持仓列表，每行 图标+名称+市值+盈亏pill，点进持仓详情页）② 现金账户（名称+余额，外币显币种）③ 实物资产（房产/黄金，名称+估值）④ 负债（信用卡显账单日/还款日+待还；贷款/借入，可折叠）。刷新按钮在净资产卡右上。
- **我的 Tab**：顶部账户卡（未登录显「登录 / 注册 ›」；已登录显头像+昵称+「会员 · 到期 2026-08-01」）→ 分组卡片：**会员订阅**(会员状态/管理订阅/恢复购买) · 外观(主题/隐藏金额) · 账户与安全(App锁/币种) · 数据与同步(iCloud/备份/导入/订阅卡片开关) · AI与扩展(API/邮箱/行情) · 关于(隐私政策/版本)。账户登录与会员订阅对接后台会员系统（见 BACKEND.md 的 Member/Plan/Subscription/登录校验 API）。

## 9. 设计优化 v4（UI 设计师评审）
> 在 v3 排版冻结基础上做视觉精修，不改动信息架构（4 底 Tab / 无标题栏 / 记账无顶分段 / 统计含「周」/ 我的含登录+会员订阅）。目标是统一视觉语言、强化层级、提升质感与可达性。

### 9.1 统一设计 Token（新增）
- **圆角**：sm 10 / md 14 / lg 16 / xl 20（列表行 12、图标块 14、卡 16、Sheet 20）。
- **阴影三档**：sh-1 `0 1px 2px rgba(0,0,0,.05)`（卡片基础）· sh-2 `0 6px 18px rgba(0,0,0,.06)`（卡片悬浮）· sh-3 `0 10px 28px rgba(0,0,0,.10)`（浮层/弹窗）。
- **间距**：4/8/12/16/20/24 基准；列表行上下 11–14，卡间距 11–12。
- 语义色沿用 v3（支出红/收入绿全局一致），新增 `color-mix` 派生浅底色用于 mini 块/盈亏 pill/选中态。

### 9.2 关键屏优化
- **记一笔（v5 金额居中）**：金额改为 hero 排版（货币符号小 + 48pt 大数字，按类型上色），**水平居中于 Pad**；分类选中态加 accent 描边 + 轻微上移；键盘键高 ≥48pt；保存键渐变。
- **导航变更（v5）**：底 Tab 由 4 项改为 明细/统计/资产/我的；记账不再占 Tab，改由明细页「+」浮钮唤起记一笔底部 Sheet（左上「取消」）。
- **明细**：汇总卡改为「结余大数字 hero + 收入/支出 mini 色块（语义浅底）」，层级更清晰。
- **统计**：卡片头部加小图标方块；趋势折线加**渐变面积填充**；拖拽手柄更明显。
- **资产**：净资产卡改 accent 渐变底 + 边框；盈亏改为圆角 pill（语义浅底）。
- **我的**：账户卡改 accent 渐变底 + 头像 accent 描边；会员分组新增「升级到会员版」CTA 按钮。

### 9.3 可达性（WCAG AA）
- 正文/标题对比度 ≥4.5:1；语义红/绿在浅暗主题下均校验可读。
- 触控目标 ≥44pt（键盘键、Tab、分类块、设置行）。
- 金额等宽数字（tabular-nums）对齐；尊重系统动态字体与减少动态。

## 10. 资产模块层级化重构（v5.1 · 第一性原理）
> 原持仓平铺 section 与账户脱节，看不到「账户下有什么、各自表现、随时间变化」。以「账户」为唯一脊柱，持仓为账户子集，形成 账户→总计→单独 三级钻取。

### 10.1 三级钻取
- **L0 净资产 hero**：总资产 − 总负债，含迷你走势 sparkline；右上刷新按钮。
- **L1 账户卡（按类别分区：投资 / 现金 / 实物 / 负债）**
  - 投资账户：显「总计」(市值/成本/盈亏%) + 嵌套持仓列表（可展开/收起）；每行 图标+名称+市值+盈亏 pill。
  - 现金/负债账户：值由「期初余额 + 交易流水净额」自动推导显余额/待还；实物账户（房产/黄金/现金外币）：手动填当前估值；信用卡显账单日+还款日。
- **L2 持仓行**：图标 + 名称 + 市值 + 盈亏 pill；点进 L3。
- **L3 持仓详情页**：见 10.2。

### 10.2 持仓详情页（L3）
- 头部：名称+代码、现价、数量、市值、成本、未实现盈亏+收益率%；另列 **已实现盈亏**（来自卖出）。
- **成本规则**：卖出成本默认**加权平均**（每笔加仓更新均价），设置可切 **FIFO 先进先出 / 手动指定批次**；已实现盈亏=卖出收入−对应成本；未实现盈亏=当前市值−剩余成本。
- 时间分段：**[周 | 月 | 季 | 年 | 总计]**，默认「月」。
  - 周=近 12 周(周点) · 月=近 12 月(月点) · 季=近 8 季(季点) · 年=近 5 年(年点) · 总计=全期(自适应降采样 日→周→月)。
- 走势图（Swift Charts）：市值线(accent) + 成本线(灰虚线) + 盈亏面积(盈绿/亏红半透明)。数据源=每日 HoldingSnapshot 降采样。
- **占比**：该持仓占所属账户 % / 占总资产 %。
- **期间汇总表**：期初市值 / 期末市值 / 期间变动 / 期间盈亏（随分段切换）。
- **交易流水**：该持仓的买入/卖出/分红列表（日期+数量+金额+类型标签）。

### 10.3 数据模型影响
- `Holding.关联账户` 改为**必填**（账户归属，无跨账户汇总；同票多账户各自独立）。
- 新增 `HoldingSnapshot(holdingID, date, qty, marketValue, costValue, unrealizedPnl)`：**全量保留不自动聚合丢弃**；走势展示时按分段自适应降采样（日→周→月）；安装前无数据，总计从安装日起。
- 实物/现金账户仍为 `Account`（type 区分），不强制含持仓。
- 资产页分区顺序固定：投资账户 → 现金账户 → 实物资产 → 负债。
